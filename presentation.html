    03The one decision everything hangs on

    Split deterministic guardrails from probabilistic judgement — and never mix them.

    Must be TRUE → code + hooks

  • Schema contracts, null/range/uniqueness
  • Referential integrity
  • Curated write-safety → PreToolUse blocks it

    • A guardrail that fires “usually” is not a guardrail.

    A JUDGEMENT → isolated + eval-gated

  • “Are these the same person?”
  • One scored function: resolve.score_pair
  • Gated by a scorecard with precision/recall

    • A fuzzy call frozen into rigid code can never improve.

    ADR-0005. Once we drew this line, every later decision had an obvious home.

    06The Scorecard — the eval caught Claude’s own bugs

    3 real matcher flaws were found by make eval failing — not by reading code:

  • reordered names missed (“Maria Hernandez” vs “Hernandez Garcia Maria J”)
  • templated email caused a false-positive merge
  • German phone format broke a true match

    • Each fix had to move a number in the scorecard, or it didn’t ship.
